# Project Rules

# Project Overview
Это архив документов на FastAPI (бек) + Vue3/Vuetify (фронт, без сборки — чистые HTML-фрагменты). Auth и работа с документами сейчас во многом заглушки (mock), а не реальная логика — это ключевой момент для дальнейшей работы. 

Функционал сайта:
- Пользователь может входить и выходить из аккаунта
- Пользователь с соответствующими правами может создавать новые документы
- Пользователь с соответствующими правами может добавлять файлы в документ
- При изменениях пользователя создается отдельная ветка документа, в которой хранятся изменения
- Пользователь с соответствующими правами может проводить review и применять изменения другого пользователя

# Project Structure
- app - backend - серверная часть
- app/api - routers - роутеры
- routers в fastapi должны быть максимально краткими и должны быть только прослойкой между FastApi и service
- app/service - логика, которая работает процессе сервера
- app/database - база данных на postgresql, может использоваться несколькими сервисами сразу
- ui - frontend
- папки и остальные каталоги в корне проекта - рабочие файлы, которые должны читаться и изменяться только по запросу

# Do
- frontend должен быть полностью основан на общении c backend
- frontend и содержимое папки ui - полная ответственность Агента
- Агент может менять ui, если меняется backend
- Mock логика должна создаваться не на уровне api, а на уровне service
- При операциях по изменению документов - document, нужно проверять, что пользователь авторизован и имеет соответствующие права
- При изменении backend Агент должен проверять, что операции не будут блокировать сервер
- При изменениях backend сначала должен создаваться план изменений, затем применяться изменения с MOCK логикой. Без прямого запроса должны создаваться только функции, которые принимают нужные аргументы и возвращают результат - заглушку
- Каждый метод Api должен использоваться в ui, проверяй это при добавлении новых методов
- Если измененный метод Api в ui Не используется, об этом нужно сообщить в Api.UsageStats.md
- Если исходный код был изменен извне, Агент не должен откатывать эти изменения

# Don't
- backend - агент не может менять без прямого указания, но может предлагать свои изменения
- Агент не может без запроса менять backend, если меняется ui
